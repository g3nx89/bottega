# Form Rubric

Calibrated rubric for form and input-centric design evaluations. Used by the `design_crit` and `iteration_delta` evaluators.

## Dimensions

### Intent Match (does the design respond to the brief?)
- **3**: Input fields and button present but no connection to the context (e.g. banking app should feel trustworthy)
- **5**: Functional form with labels and fields, but generic — could be any app
- **7**: Form communicates the right context — a banking login feels secure, a creative app login feels playful
- **9**: Form with contextual refinement — brand-specific elements, tone-appropriate micro-copy implied by layout choices

### Visual Craft (is it curated?)
- **3**: Default inputs, no visual treatment, labels awkwardly placed
- **5**: Styled inputs with consistent borders, reasonable spacing
- **7**: Refined inputs — subtle focus states implied, consistent border radius, well-proportioned label-to-input ratio, clear visual grouping
- **9**: Polished craft — input states differentiated, error/success patterns implied, balanced density, optical alignment of labels and fields

### Design Decisions (are choices intentional and reasoned?)
- **3**: Default styling, no thought about trust or usability signals
- **5**: Some intentional choices (brand color on submit button, grouped related fields)
- **7**: Clear UX decisions — password field has appropriate visual weight, primary action distinguished from secondary, field grouping reflects mental model
- **9**: Sophisticated decisions — progressive disclosure implied, visual hierarchy guides completion order, trust signals (lock icon, security mention) integrated naturally

### Hierarchy (does the visual hierarchy guide the eye?)
- **3**: All fields look identical — no clear starting point or completion path
- **5**: Submit button is visually distinct, but field ordering feels arbitrary
- **7**: Clear completion path: title > fields in logical order > primary action. Labels guide without cluttering
- **9**: Effortless flow — every field has the right visual weight for its importance, grouping and spacing create natural reading rhythm

### Consistency (is it cohesive?)
- **3**: Mixed input styles, inconsistent spacing, label alignment varies
- **5**: Consistent input styling but doesn't feel systematic
- **7**: All inputs share a consistent system — same border radius, spacing rhythm, label placement, color roles
- **9**: Design system quality — inputs, labels, buttons, and spacing form a cohesive system that could extend to any form in the app
