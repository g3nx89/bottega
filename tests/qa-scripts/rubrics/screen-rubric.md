# Screen Rubric

Calibrated rubric for full-screen / settings / dashboard design evaluations. Used by the `design_crit` and `iteration_delta` evaluators.

## Dimensions

### Intent Match (does the design respond to the brief?)
- **3**: Content areas present but layout doesn't reflect the screen's purpose
- **5**: Functional screen with expected sections, but generic — could be any app's settings
- **7**: Screen communicates its specific context — a dark music player feels immersive, a settings screen feels organized and scannable
- **9**: Screen with contextual mastery — layout, density, and visual treatment perfectly match the use case and expected user mental model

### Visual Craft (is it curated?)
- **3**: Content dumped into areas with no visual treatment, inconsistent spacing
- **5**: Reasonable layout with consistent spacing, basic section separation
- **7**: Refined layout — clear section hierarchy, appropriate density, consistent gutters, subtle separators or grouping
- **9**: Polished craft — visual rhythm across sections, considered whitespace, refined typography scale, subtle depth cues

### Design Decisions (are choices intentional and reasoned?)
- **3**: Default layout, no consideration for information density or scan patterns
- **5**: Some intentional grouping, a nav element, basic section headers
- **7**: Clear information architecture — sections grouped logically, navigation supports the mental model, density appropriate to content type
- **9**: Sophisticated decisions — responsive density, progressive disclosure implied, navigation hierarchy matches task frequency, theming consistent with brand

### Hierarchy (does the visual hierarchy guide the eye?)
- **3**: Everything at the same level — flat list of options with no visual organization
- **5**: Section headers provide some structure, but within sections everything is flat
- **7**: Clear multi-level hierarchy: navigation > section headers > content items > secondary controls. Scan path is natural
- **9**: Sophisticated hierarchy — primary actions discoverable, secondary options accessible but not competing, grouping creates scannable patterns

### Consistency (is it cohesive?)
- **3**: Mixed styles across sections, inconsistent spacing, alignment varies
- **5**: Each section is internally consistent but sections don't share a system
- **7**: All sections share a consistent design language — same spacing grid, type scale, color roles, interactive patterns
- **9**: System-level consistency — every section could be a reusable pattern, the screen feels like part of a polished app
